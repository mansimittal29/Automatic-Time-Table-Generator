📊 Publishing Timetables

Complete workflow guide for publishing timetables to all users

Overview: The Publishing Workflow

The timetable generation system uses a three-stage approval workflow to ensure quality control before timetables are shown to students and faculty:

1. DRAFT
Just generated
→
2. APPROVED
Reviewed & OK
→
3. PUBLISHED
Live & visible
ℹ️ Draft:
A freshly generated timetable. Nobody outside of admins can see it yet.
ℹ️ Published:
The currently active timetable. Visible via API & available to all users.
  • Draft: Admin generates timetable → auto-saved as draft
  • Approve: Admin reviews & clicks "Approve" button
  • Publish: Admin clicks "Publish" → becomes live
  • Only 1 Published: Publishing a new version archives the old one
🚀 Step-by-Step Publishing Guide

Step 1: Generate a Timetable

This creates the initial DRAFT version.

  1. Go to Dashboard → Generate Timetable
  2. Fill in the form (assignments, rooms, working days, periods, constraints, etc.)
  3. Click Generate
  4. System generates the timetable and saves it as DRAFT
✅ Flash message: "Timetable generated successfully. Saved as draft version V[YYYYMMDD-HHMMSS]."

Step 2: Review the Generated Timetable

Inspect the timetable output to verify there are no conflicts or issues:

  • Check Utilization % — are resources well-used?
  • Look for Free Slots — are they acceptable?
  • Verify Lab Double Blocks — are contiguous?
  • Check Faculty Loads — are they within limits?
  • View Room Utilization — no clashes?
⚠️ If there are issues, go back and generate a new version with adjusted parameters. There will be multiple draft versions.

Step 3: Navigate to Version Management

Once you're satisfied with the timetable, go to the version approval interface.

From the timetable result page, click:

Dashboard → Compare Versions OR
Direct URL: /admin/versions

Step 4: Approve the Version

Transition from DRAFT to APPROVED:

  1. Find your timetable in the version table
  2. Click the Approve button
  3. Status changes to APPROVED (blue)
✅ Flash message: "Timetable version [V20260317-143022] has been approved."

Step 5: Publish the Version

Transition from APPROVED to PUBLISHED:

Timetable Version Workflow Table:
ID Version Label Status Score Created Actions
#5 V20260317-143022 APPROVED 87.5 2026-03-17 14:30
  1. Once approved, the Publish button appears
  2. Click Publish
  3. Status changes to PUBLISHED (green ✓)
  4. Old published versions are archived automatically
  5. Timetable is now LIVE and visible to all users
✅ Flash message: "Timetable version [V20260317-143022] has been published and is now visible to all users."

Step 6: ✨ All Users Can Now See It

Once published, all users (Faculty, Students, HODs) can access:

  • 👥 Their Personal Timetable — via the interface
  • ⚙️ REST APIs:
    • /api/timetable/latest — Full class-wise timetable
    • /api/timetable/class/latest — Class-wise view
    • /api/timetable/faculty/latest — Faculty-wise view
    • /api/timetable/room/latest — Room-wise view
  • Download Formats:
    • /download/xlsx — Excel workbook
    • /download/csv — CSV spreadsheet
    • /download/json — JSON data

🎯 Key Features of the Publishing System

✅ Only ONE Published Version at a Time

When you publish a new version, the previously published version is automatically ARCHIVED. This ensures there's always exactly one "current" timetable visible to users.

✅ Compare Multiple Drafts Before Approval

Admins can generate multiple timetables with different parameters, then compare them side-by-side before deciding which one to approve and publish.

✅ Audit Trail

Every version stores metadata:
• Created By: Which admin generated it
• Approved By: Which admin approved it
• Published By: Which admin published it
• Timestamps: When each action happened

✅ Automatic Export on Publishing

When a version is published, the system automatically exports it to:
• xlsx (Excel)
• csv (Spreadsheet)
• json (API data)

These files are saved to the output/ folder for download and API access.
❓ Common Scenarios

Scenario 1: The timetable has conflicts

Problem: I generated a timetable but it has free slots or resource clashes.

Solution:

  1. Do NOT approve it
  2. Go back and generate a new timetable with different parameters (more time periods, different assignments, etc.)
  3. Compare both at /admin/versions
  4. Approve and publish the better one

Scenario 2: I accidentally published the wrong version

Problem: I published a timetable, but it's not good.

Solution:

  1. Generate a new, better timetable
  2. Approve it to APPROVED status
  3. Publish it — the old one is automatically archived
⚠️ Note: There's no "unpublish" button — you must create and publish a new version to replace the old one.

Scenario 3: Multiple admins are generating timetables

Problem: Several admins are testing different parameters.

Solution:

  1. Each admin generates their own version
  2. Go to /admin/versions to see all drafts
  3. Compare different versions (select 2 versions to compare side-by-side)
  4. Collectively decide which is best
  5. Approve and publish the winner

🔧 Troubleshooting

Q: Why can't I see the Publish button?

A: The Publish button only appears when a version is in APPROVED status. Make sure you've clicked Approve first.

Q: Why are my drafts not showing up?

A: Make sure you're logged in as an admin or HOD. Students and Faculty cannot see draft versions. Go to /admin/versions (not the main dashboard).

Q: Can I publish multiple timetables at once?

A: No. Only one version can be published at a time. Publishing a new version automatically archives the old one.

Q: Where are the exported files (xlsx, csv, json)?

A: They're saved in the timetable-generator/output/ folder on the server. They're automatically created when you publish a version.

Q: Can students/faculty download the timetable?

A: Yes! Once published, they can download from /download/xlsx, /download/csv, or /download/json, or access via the APIs.

📝 Quick Summary
The 3-Step Publishing Process:

1. Generate ➜ Creates a new DRAFT version
2. Approve ➜ Changes status to APPROVED (admin/HOD)
3. Publish ➜ Changes status to PUBLISHED (admin only)

Once published, all users can see and download the timetable!
