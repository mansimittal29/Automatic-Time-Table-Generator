📍 Login Page

Route: /login

Sign In

🔴 Auto-Redirect Enforcement

⚠️ Important: When must_change_password = True:
  • User logs in → redirected to /change-password
  • ANY attempt to access other pages (dashboard, admin, etc.) → redirected back to /change-password
  • Flash message appears: "You must change your password before continuing."
  • Only allowed pages: /change-password, /logout, static files

🔐 Change Password Page

Route: /change-password (GET & POST)

Protection: @login_required

Change Password

⚠️ Your account requires a password change before you can continue.

Minimum 6 characters.
ℹ️ Form Validation:
  • Current password must match the user's current password hash
  • New password must be at least 6 characters
  • New password & Confirm must match
  • Passwords are hashed with werkzeug.security before storage
✅ On Success:
  • Password updated in database
  • must_change_password flag is cleared to False
  • Flash message: "Password updated successfully."
  • User redirected to Dashboard
  • User can now access all pages normally

❌ Error Scenarios

Error Condition Flash Message Action
Current password incorrect "Current password is incorrect." Stay on form, user can retry
New password < 6 chars "New password must be at least 6 characters." Stay on form, user can retry
Passwords don't match "New passwords do not match." Stay on form, user can retry
⚙️ Technical Implementation

Database Model

class User(UserMixin, db.Model): ... must_change_password = db.Column(db.Boolean, nullable=False, default=False) ...
  • Column type: Boolean
  • Default value: False
  • Automatically migrated to existing databases

Routes & Decorators

Route Method Protection Purpose
/admin/users GET, POST @roles_required("ADMIN") Create users & view list
/change-password GET, POST @login_required User changes password
/admin/users/<id>/toggle-active POST @roles_required("ADMIN") Activate/deactivate user
/admin/users/<id>/reset-password POST @roles_required("ADMIN") Admin resets user password

Request Hook: before_request

@app.before_request def enforce_password_change(): if not current_user.is_authenticated: return if getattr(current_user, "must_change_password", False): allowed_endpoints = {"change_password", "logout", "static"} if request.endpoint not in allowed_endpoints: flash("You must change your password before continuing.", "warning") return redirect(url_for("change_password"))
  • Runs on every request for authenticated users
  • Checks if must_change_password = True
  • Blocks access to all pages except password change & logout
  • Non-intrusive: only redirects if flag is set
🚀 Quick Start Guide

For Admins

  1. Log in with admin credentials (admin / admin123)
  2. Go to Dashboard → Admin Tools → User Management
  3. Fill in the "Create User" form:
    • Username (required)
    • Full Name (required)
    • Email (optional)
    • Temporary Password (required, min 6 chars)
    • Role (select from dropdown)
  4. Click Create User
  5. Share username + password with user
  6. Optional: Use Reset Pwd or Deactivate/Activate buttons as needed

For New Users (First Login)

  1. Navigate to login page: /login
  2. Enter username provided by admin
  3. Enter temporary password
  4. Click Sign In
  5. You will be automatically redirected to Change Password
  6. Fill in the password change form:
    • Current Password (the temp password you received)
    • New Password (your choice, min 6 characters)
    • Confirm New Password
  7. Click Update Password
  8. You're now logged in and can access the dashboard normally!
💡 Pro Tips:
  • Admins can reset any user's password anytime via the "Reset Pwd" button
  • Deactivating a user doesn't delete them—just prevents login
  • The "Pwd Change?" column makes it easy to see who hasn't set their own password yet
  • Users CANNOT bypass the password change—the before_request hook enforces it
