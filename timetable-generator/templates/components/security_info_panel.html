🔒 Security Configuration

Password Policy
  • Minimum Length: 10 characters
  • Allowed Characters: All printable ASCII
  • Format Rules: No leading/trailing spaces
  • Reuse: Cannot reuse current password (change-password only)
Login Protection
  • Rate Limiting: 5 failed attempts per 15 min window
  • Lockout Duration: 10 minutes after threshold
  • Tracking: By IP address and username (scoped)
  • Recovery: Contact administrator if locked out
Session Security
  • Cookies: HttpOnly, Secure, SameSite=Lax
  • CSRF Protection: Same-origin enforcement
  • Timeout: Browser session lifetime (or configured limit)
Request Validation
  • Origin Checks: Cross-origin POST/PUT/PATCH/DELETE blocked
  • Headers: X-Content-Type-Options, X-Frame-Options set
  • Referrer Policy: Strict-origin-when-cross-origin

ℹ Environment Variables: All thresholds can be customized via environment variables with TIMETABLE_ prefix (e.g., TIMETABLE_MIN_PASSWORD_LENGTH, TIMETABLE_LOGIN_MAX_FAILURES). See README for complete list.
